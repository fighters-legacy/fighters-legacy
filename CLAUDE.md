# Fighters Legacy ‚Äî Claude Code Instructions

## Project Overview

GPL v3 general-purpose combat flight sim engine, inspired by Jane's Fighters Anthology (1998).
Cross-platform: Windows 10/11, Linux, macOS. Phase 2 (Modern-Particles Engine) is active.

## Architecture

```
engine/         ‚Äî core: content system, asset manager, IContentPack interface, security hardening
engine/entity/  ‚Äî entity/object system: pool, type registry, damage model, EntityManager
engine/net/     ‚Äî authoritative server sim: WorldBroadcaster, GameProtocol wire types
engine/perf/    ‚Äî performance overlay: PerformanceOverlay
engine/render/  ‚Äî sim‚Üírender bridge + scene submission: RenderSnapshot, SimRenderBridge, SceneRenderer, CameraController, ParticleSystem
engine/script/  ‚Äî Lua 5.4 sandbox: LuaSandbox (engine-script library); deny-listed io/os/package/debug; custom require restricted to pack ai/ dir
engine/weather/ ‚Äî server-authoritative weather state machine, time-of-day clock, wind/gust/turbulence model: WeatherController (engine-weather library)
platform/       ‚Äî HAL: Vulkan, SDL3, OpenAL Soft, ENet backends
platform/RenderTypes.h ‚Äî GPU-agnostic scene types shared across the HAL boundary
server/         ‚Äî dedicated server binary
server/fl-server/ ‚Äî fl-server: authoritative headless game server (owns GameLoop + EntityManager)
game/           ‚Äî fighters-legacy game binary (ENet client connecting to embedded fl-server in single-player)
tools/          ‚Äî developer utilities; asset pipeline (validate-flight-model, validate-mission, validate-licenses, validate-mesh, tex-compress); net_check (ENet smoke-test); blender_gen.py; gen_terrain_chunks.py
tests/          ‚Äî Catch2 unit tests (C++); pytest unit tests for Python tools (test_gen_terrain_chunks.py)
```

The engine is fully content-agnostic. It knows nothing about FA or any specific game.
FA support lives in jomkz/fa-content. No FA-specific code belongs in this repo.

`ft-gui` is an old name for `fighters-codex` (a separate project). It has no role here. Phase 4 UI work follows the `IGui` HAL pattern (pure-virtual interface + swappable backends, same as `IRenderer`); no ft-gui references belong in this repo.

### Math library

**GLM** is the shared vector/matrix/quaternion library, linked as an INTERFACE dependency on `platform-hal`. Anything that links `platform-hal` (engine, game, tests) gets GLM automatically. Use `glm::vec3`, `glm::mat4`, `glm::quat`, etc.

### Renderer architecture (Phase 2)

`VkRenderer` (platform/vulkan/) uses Vulkan 1.3 dynamic rendering (`VK_KHR_dynamic_rendering`) ‚Äî no `VkRenderPass` or `VkFramebuffer` objects. Seven steps per frame (one compute dispatch + six rendering scopes):

1. **Shadow** ‚Äî `kNumCascades=4` PSSM cascades rendered into a `kShadowRes=2048` 2D depth array (`VK_FORMAT_D32_SFLOAT`, **forward-Z**, depth clear = 1.0). Cascade matrices computed via tight bounding-sphere fit; PCF comparison sampler (`VK_COMPARE_OP_LESS_OR_EQUAL`). `ShadowUBO` bound at set 0, binding 2; `sampler2DArrayShadow` at set 0, binding 3.
2. **Particle compute** ‚Äî `particle_sim.comp` dispatched before forward pass; advances age/pos for `kMaxParticles=8192` slots (local_size_x=64). New particles written to a host-visible spawn staging buffer by CPU each frame then `vkCmdCopyBuffer`'d into the device-local pool SSBO (ring-buffer overwrite). Push constant `{dt, count, gravity, _pad}` (16 bytes). Barrier: COMPUTE_WRITE ‚Üí VERTEX_SHADER_READ before forward pass.
3. **Forward (opaque)** ‚Äî Cook-Torrance PBR (GGX NDF, Smith geometry, Schlick Fresnel) with normal maps + ORM textures (set 1: base color / normal / ORM at bindings 0‚Äì2). Geometry into HDR offscreen (`VK_FORMAT_R16G16B16A16_SFLOAT`) with **reverse-Z** depth (`VK_FORMAT_D32_SFLOAT`, far = 0.0, depth clear = 0.0, compare = GREATER). Depth storeOp = STORE (sky + particle + transparent + bloom passes read depth).
4. **Sky + particles** ‚Äî combined rendering scope: sky fullscreen triangle (GREATER_OR_EQUAL, depth write off) followed by particle billboard draw (GREATER, depth write off, depth test on). Two instanced draws (`vkCmdDraw(6, kMaxParticles, 0, 0)`): additive pipeline (fire/explosion) then alpha pipeline (smoke). `particle.vert` reads particle SSBO and camera UBO from set 0 (bindings 0‚Äì1); push constant `uint32_t renderAdditive` selects which blend-mode particles to emit (others output degenerate off-screen positions).
5. **Transparent** ‚Äî alpha-blended items (materials with `MaterialDesc::alphaBlend=true`) drawn back-to-front using `m_forwardAlphaPipeline` (depth write OFF, cull NONE, SRC_ALPHA/ONE_MINUS_SRC_ALPHA blend). Sorted in `recordCommandBuffer` by descending squared camera distance.
6. **Bloom** ‚Äî enabled when `RendererSettings::bloom=true`. Three half-resolution passes into `m_bloomImage`/`m_bloomAuxImage`: bright-pass threshold (`bloom_threshold.frag`), horizontal Gaussian blur, vertical Gaussian blur. `TonemapPush::bloomStrength` controls additive blend weight in the tonemap shader.
7. **Tonemap** ‚Äî Khronos PBR Neutral + optional bloom composite (binding 1 = bloom buffer) + optional FXAA (5-tap luma edge-detect + 3-sample blur). `TonemapPush` (16 bytes): texelSizeX/Y, enableFxaa, bloomStrength. Fullscreen HDR ‚Üí swapchain (`B8G8R8A8_SRGB`).

**Note:** shadow passes use forward-Z (near=0, far=1); scene depth uses reverse-Z. These are independent depth spaces.

World convention: right-handed, Y-up, meters (matches glTF). Vulkan clip-space Y-flip handled in the projection matrix. **World positions are `double`/`glm::dvec3`** throughout the engine (`EntityTransform::pos`, `MsgEntityEntry::pos`, `EntityRenderEntry::position`, `CameraView::worldOrigin`). Camera-relative rendering subtracts `dvec3 worldOrigin` before GPU upload, casting the small relative offset to `vec3` ‚Äî float32-safe at any scale including planet-scale.

**Texture upload:** KTX2 Basis Universal ‚Üí BC7 (desktop, if `VK_FORMAT_BC7_UNORM_BLOCK` supported) ‚Üí ASTC 4√ó4 (Apple Silicon, if BC7 absent) ‚Üí RGBA32 fallback. All mip levels uploaded via `createGpuImageCompressed` using `ktxTexture_GetImageOffset` per mip. sRGB/UNORM views chosen per texture semantic (base color = sRGB, normal/ORM = UNORM). Normal maps use tangent-space flat normal default `{128,128,255}`; ORM defaults to all-ones linear white.

Runtime shader discovery: `VkRenderer::resolveShaderDir()` tries `SDL_GetBasePath()` + `"shaders/"` first, then macOS `.app` bundle path, then the build-tree `FL_SHADER_DIR` fallback. Release packages must stage `*.spv` into `dist/shaders/` (see `release.yml`).

**Renderer instantiation:** game and tool code must use `createVulkanRenderer()` from `platform/vulkan/VkRendererFactory.h` ‚Äî never include `VkRenderer.h` directly. `VkRenderer.h` pulls in `VkResources.h` ‚Üí `vk_mem_alloc.h`, which is only on the private include path of `platform-vulkan`.

**GPU UBO / push constant sizes (after #39):** `CameraUBO` = 144 bytes (set 0 binding 0); `LightUBO` = 64 bytes (set 0 binding 1; +`fogParams` vec4 for density/startDist/timeOfDay); `ShadowUBO` = 272 bytes (set 0 binding 2). `SkyPushConstants` = **128 bytes exactly** ‚Äî fills the Vulkan-guaranteed minimum push constant budget. Do NOT add more fields to `SkyPushConstants` without first migrating the sky pass to use a UBO. `ForwardPushConstants` = 96 bytes. Sky pass uses FBM procedural clouds driven by `skyParams.w` (cloudCoverage) and `fogParams` (density/startDist/timeOfDay).

**GLM extension headers:** `VkRenderer.cpp` requires `<glm/gtc/matrix_transform.hpp>` (for `glm::lookAt`) and `<glm/ext/matrix_clip_space.hpp>` (for `glm::orthoZO`). `engine/render/RenderSnapshot.h` and `engine/entity/EntityManager.cpp` require `<glm/gtc/quaternion.hpp>` (for `glm::quat`). These are not in `<glm/glm.hpp>` core ‚Äî always include them explicitly.

### Server-client architecture (Phase 2, feat/sandbox-server-client)

**Single-player**: the game client spawns the `fl-server` binary as a subprocess bound to `127.0.0.1:4778` (`LocalServer` in `game/fighters-legacy/`), then connects to it as a normal ENet client ‚Äî no server infrastructure code lives in the game binary. `fl-server` is the identical dedicated server used for multiplayer. `LocalServer::start()` generates a 24-char hex session token, then spawns fl-server with `[port] 1 --bind <addr> --admin-token <token>` and waits for the "listening on" log line before returning.

**Game-layer classes** (in `game/fighters-legacy/`):
- `LocalServer` ‚Äî subprocess launcher; wraps `platform/Subprocess`; finds fl-server via `FL_SERVER_DEV_PATH` (CMake-injected dev-build path) or `SDL_GetBasePath()` (release co-located). `start()` generates a 24-char hex session token and passes it via `--admin-token`; `sessionToken()` returns it for use by `makeNetworkAdminSender` in `main.cpp`. `registerDebugCommands(registry, serverCommand, ...)` wires the pre-built `serverCommand` function (a `MsgAdminCommand` sender) into `DebugCommandContext`.
- `ClientNetEventHandler` (`ClientNetEventHandler.h`) ‚Äî `INetworkEventHandler` for the game client; parses all message types and calls `bridge.publishExternal()` + `tickAlpha.markNewTick()` on each `WorldSnapshot`. Holds `ClientTickAlpha tickAlpha` for render interpolation (see below).
- `CameraInput` ‚Äî owns the 13 previously loose camera orbit state variables; translates SDL keyboard/mouse input to `CameraController` updates.
- `GameHud` ‚Äî composites `FlightHud` + server-notice banner into a single HUD layer. **Fully decoupled from `DebugConsole`** ‚Äî the debug console is an independent overlay available in all game states including the future main menu. Callers merge `GameHud::buildElements()` and `dbgConsole.elements()` before the single `submitHudElements` call.

**Render alpha**: previously computed by `serverLoop.shellTick()` (in-process sim thread). Now computed client-side by `ClientTickAlpha::get()` ‚Äî a wall-clock timer reset on each received `MsgWorldSnapshot`. No server infrastructure needed.

**Debug commands** (`spawn`/`kill`/`tp`/`set_weather`): `DebugCommandContext::serverCommand` (a `std::function<void(std::string_view)>`) sends a `MsgAdminCommand` (0x06) over ENet using the per-session token (`makeNetworkAdminSender` in `game/fighters-legacy/main.cpp`). In single-player the token is generated by `LocalServer::start()` and passed to fl-server via `--admin-token`. `nullptr` in multiplayer until the connection UI exists (future issue). The server admin console also has a `tp <idx> <x> <y> <z>` command.

**Platform layer**: `platform/Subprocess` (`Subprocess.h/.cpp`) provides cross-platform child process management (`CreateProcess`/`posix_spawn`, stdin/stdout pipes, liveness check, graceful+forceful stop). All `#ifdef WIN32` guards are confined to `Subprocess.cpp`; callers are `#ifdef`-free. `platform/StdoutLogger.h` is a shared header-only `ILogger` implementation for headless binaries.

**Shared engine utilities** (added in this refactor): `engine/config/ConfigFile.h` (`fl::ensureAndReadConfig`, `fl::writeConfigFile` ‚Äî raw-fstream config file helpers used by fl-server); `engine/net/NetworkUtils.h` (`fl::normalizeIp` ‚Äî IP normalisation previously duplicated in WorldBroadcaster.cpp and fl-server).

- **`engine/net/GameProtocol.h`**: packed wire types (`#pragma pack(1)`). Always parse with `std::memcpy` ‚Äî never cast raw packet pointers. `kProtocolVersion = 1` (uint16_t). Eleven message types: `MsgHello` (4 bytes, `MsgId::Hello = 0x00`, server‚Üíclient version handshake), `MsgConnectAck` (12 bytes, includes `assignedEntityIdx/Gen`), `MsgEntityTypeDef` (196 bytes), `MsgWorldSnapshotHeader` (12 bytes, offset 1 = `protocolVersion uint8_t`), `MsgEntityEntry` (68 bytes, double pos), `MsgClientInput` (44 bytes, `MsgId::ClientInput = 0x03`, offsets 2‚Äì3 = `protocolVersion uint16_t`), `MsgWeatherState` (20 bytes, `MsgId::WeatherState = 0x04`, server‚Üíclient unreliable ~6 Hz; carries `preset uint8_t`, `timeOfDayTenths uint16_t` [hours√ó10], `fogDensity`, `fogStartDist`, `windX`, `windZ` ‚Äî additive ID, old clients silently discard, no version bump), `MsgServerNotice` (64 bytes, `MsgId::ServerNotice = 0x05`, server‚Üíclient reliable; `secondsRemaining uint16_t` at offset 2 [0 = shutdown imminent], `text char[60]` at offset 4; additive ID, old clients discard, no version bump), `MsgAdminCommand` (128 bytes, `MsgId::AdminCommand = 0x06`, client‚Üíserver reliable; `token char[30]` at offset 2, `command char[96]` at offset 32; additive ID, old servers discard), `MsgAdminResponse` (128 bytes, `MsgId::AdminResponse = 0x07`, server‚Üíclient reliable; `text char[126]` at offset 2; additive ID, old clients discard), `MsgLanBeacon` (74 bytes, `MsgId::LanBeacon = 0x10`, raw UDP broadcast not sent over ENet). `kGameModeCampaign/Mission/Sandbox` bit constants for `MsgLanBeacon::gameModeFlags`. Wire quaternion order: `x, y, z, w` matching `EntityTransform::quat`. See `docs/network-protocol.md` for field tables.
- **`engine/net/WorldBroadcaster`**: `ISimUpdate` + `INetworkEventHandler`. `onConnect` check order: (1) ban check (`m_bannedAddresses` ‚Äî disconnect + return if banned), (2) allowlist check (`m_allowedAddresses` ‚Äî if non-empty and IP not listed, disconnect + return), (3) rate-limit check (sliding window per IP using `m_connectRecords` ‚Äî if size > `m_connectRateLimit`, disconnect + return); then sends `MsgHello` (reliable), spawns `"builtin:debug-entity"` for the peer, creates a `FlightIntegrator` stored in `m_peerFlightSims`, and sends `MsgConnectAck` + `MsgEntityTypeDef[]`. `onDisconnect` kills the entity and erases peer from all maps including `m_peerFloodState`. `onReceive` checks `MsgClientInput` flood rate in a 1-second window (`m_peerFloodState`; threshold = `60 * m_floodMultiplier`), then validates `protocolVersion` and stores ‚Üí `PeerInputState`. `onTick` steps each peer's `FlightIntegrator`, calls `EntityManager::onTick`, builds and broadcasts `MsgWorldSnapshot`, broadcasts `MsgWeatherState` every 10 ticks (~6 Hz), drives shutdown countdown (broadcasts `MsgServerNotice` at each warning interval + T-60s clamp + T=0, then invokes `m_shutdownCallback`), and prunes stale `m_connectRecords` entries every 600 ticks. Constructor takes optional `WeatherController* weather = nullptr`. `engine-net` privately links `engine-flight` and `engine-weather`. `getPeerCount() const noexcept` ‚Äî reads `std::atomic<int> m_activePeerCount`; safe to call from any thread. **Peer management (sim-thread only):** `kickPeer(peerId)` ‚Äî calls `INetwork::disconnectPeer`; `banAddress(ip)` ‚Äî normalises IP, inserts into `m_bannedAddresses`, kicks matching peers; `unbanAddress(ip)` ‚Äî erases from ban set; `forEachPeer(fn)` ‚Äî iterates `m_peerEntities`, copies each `getPeerAddress` string per entry; `setBannedAddresses(set)` ‚Äî replaces entire ban set (used at startup to load banlist file); `setAllowedAddresses(set)` ‚Äî replaces allowlist (empty = disabled); `getBannedAddresses()` ‚Äî returns copy of current ban set; `setRateLimitParams(maxConnects, windowSeconds, floodMultiplier)` ‚Äî configure rate limiting before `gameLoop.start()`; `setClockOverride(fn)` ‚Äî inject a clock function (tests only); `setOperatorPassword(pw)` ‚Äî empty = admin channel disabled; `setAdminDispatch(fn)` ‚Äî `std::function<std::string(std::string_view)>` wrapping `adminRegistry.dispatch`; both call before `gameLoop.start()`. **Shutdown countdown (sim-thread only):** `initiateShutdown(secondsDelay, warningIntervalS)`, `cancelShutdown()`, `extendShutdown(additionalSeconds)‚Üíbool` (false if not active), `isShuttingDown()`, `secondsUntilShutdown()`; `setShutdownCallback(fn)` ‚Äî must be called before `gameLoop.start()`; callback invoked at T=0 on sim thread (fl-server wires it to set `g_quit=1`). **`onReceive`** handles `MsgClientInput` (flood-rate-checked, stored ‚Üí `PeerInputState`) and `MsgAdminCommand` (0x06) (constant-time token comparison against `m_operatorPassword`; dispatches via `m_adminDispatch(cmd)`; sends `MsgAdminResponse` reliable unicast to requesting peer); unknown msgIds silently discarded. Private: `m_operatorPassword`, `m_adminDispatch` (network admin channel state).
- **`ClientNetEventHandler`** (`game/fighters-legacy/ClientNetEventHandler.h`): parses `MsgHello` (checks `protocolVersion == kProtocolVersion`, calls `net.disconnect()` on mismatch), `ConnectAck` (stores `assignedEntityIdx/Gen`), `WorldSnapshot` (calls `bridge.publishExternal()` + `tickAlpha.markNewTick()`), `WeatherState` (updates `EnvironmentState`), `ServerNotice` (calls `console->print()` + `hud->setNotice()`), `AdminResponse` (0x07) (calls `console->print()` with `[admin]` prefix if text non-empty). Sends `MsgClientInput` each frame (arrow keys + shift + space).
- **ENet threading rule**: server `ENetHost*` is owned exclusively by fl-server's sim thread (via `WorldBroadcaster::onTick ‚Üí net->service(0)`); client `ENetHost*` by the game's main thread. Never cross-thread.
- **`GameLoop::enqueueSimCallback(fn)`**: thread-safe lambda queue drained at the top of each sim tick before `ISimUpdate::onTick()`. Used by fl-server admin commands to mutate `EntityManager` safely. Debug commands in the game client forward to fl-server's admin console via `serverCommand` callback instead.
- **Single-player render alpha**: computed by `clientHandler.tickAlpha.get()` ‚Äî `ClientTickAlpha` is a wall-clock timer reset on each received `MsgWorldSnapshot`. Replaces the removed in-process `serverLoop.shellTick()`.

### Sim‚Üírender bridge (Phase 2, PR 4)

`engine/render/SimRenderBridge` is a **lock-free triple-buffer** that ships a per-tick entity snapshot from the sim thread to the render thread. Three `RenderSnapshot` slots rotate: one owned by the sim, one in the atomic spare, one held by the render thread. `publish()` moves the completed snapshot into the spare (release fence); `tryAdvance()` atomically swaps the render slot with the spare when a newer tick is available (acq_rel fence). All three slot indices are always a distinct permutation of {0,1,2}.

- `EntityRenderEntry`: entityIdx, entityGen, typeIndex, position (**glm::dvec3** ‚Äî double-precision world position), orientation (glm::quat, w-first constructor), velocity (glm::vec3 for sub-tick extrapolation), damageLevel (uint8_t), playerOwned.
- **`publishExternal(RenderSnapshot)`** ‚Äî main-thread-only variant for network-client mode (no concurrent sim thread). Used by `ClientNetEventHandler` after parsing a `WorldSnapshot` packet.
- In server-client mode, do NOT call `EntityManager::setRenderBridge()` ‚Äî `WorldBroadcaster` owns serialisation; the render bridge is populated from network packets instead.
- `engine-render` CMake library is **unconditional** (no Vulkan dep) ‚Äî builds in CI without a GPU. `engine-entity` privately links `engine-render`; `engine-render` privately links `engine-content`. Any binary/test that links `engine-entity` gets both resolved automatically.

### Scene renderer (Phase 2, PR 5)

`engine/render/SceneRenderer` converts the per-tick entity snapshot into a `FrameScene` and calls `IRenderer::setScene` each frame. All work is main-thread.

- **MeshNameResolver**: `std::function<bool(uint32_t typeIndex, std::string& mesh, std::string& dmg)>` ‚Äî injected at construction to avoid a circular CMake dep between `engine-render` and `engine-entity`. In `main.cpp` the lambda captures `EntityTypeRegistry&`.
- **EffectResolver**: `std::function<std::string(uint32_t typeIndex, uint8_t damageLevel)>` ‚Äî optional; injected via `setParticleSystem(ParticleSystem*, EffectResolver)`. Called for each entity with `damageLevel > 0`; returns the particle preset name (e.g. `"explosion"`) or empty string. In `main.cpp` reads `EntityDef::damage` through the registry.
- **Mesh/material cache**: `getOrUploadMesh` / `getOrUploadMaterial` call `IRenderer::createMesh` / `createMaterial` once per unique mesh name; subsequent frames use the cached handle.
- **Camera-relative rendering**: `glm::vec3 relPos = glm::vec3(dvec3(entry.position) - camera.worldOrigin)` ‚Äî subtracts two `dvec3` values and casts the small result to `vec3` for GPU upload. Safe at planet scale.
- **Velocity extrapolation**: `rendered_pos = entry.position + entry.velocity * (alpha * kTickDt)` where `alpha = clientHandler.tickAlpha.get()` (client-side wall-clock alpha) and `kTickDt = 1/60 s`.
- **Damage variant**: if `entry.damageLevel > 0` and `EntityDef::classicDamageMesh` is non-empty, the damage mesh is loaded instead; `kRenderFlagDamaged` is set on the `RenderItem`.
- **Sort**: opaque items sorted front-to-back by squared camera-relative distance to minimise overdraw.
- **Draw-distance cull**: `setDrawDistance(float km)` configures the entity cull radius; entities beyond it are skipped before RenderItem construction. Wired from `RendererSettings::drawDistanceKm` in `main.cpp`.
- **Builtin palette fallback**: when `MeshNameResolver` returns false, `meshName` is empty, or mesh upload fails, the entity is rendered with a builtin tetrahedron (~10 m, +X forward) from a 6-color palette cycled by `entityIdx` (0‚Äì2 opaque, 3‚Äì5 glass/transparent). Palette and meshes are uploaded once on first `renderFrame` call via `ensureBuiltins()`. `setBuiltinFloor(true)` appends a 4 km flat floor plane as the final opaque item. Both are defined in `engine/render/BuiltinGeometry.h`.
- `SceneRenderer::renderFrame(alpha, camera, env, extraEmitters={})` calls `tryAdvance()` internally; callers must NOT also call `renderBridge.tryAdvance()`. `extraEmitters` is merged with entity-damage effects when a ParticleSystem is set.

### GPU particle system (Phase 2, PR 6)

`engine/render/ParticleSystem` manages per-frame emitter emission on the CPU. `VkRenderer` handles GPU simulation and rendering.

- **ParticlePreset** (`engine/render/ParticleSystem.h`): spawnRate, particleLifetime, initialSpeed, colorStart/End, sizeStart/End, additive flag.
- `ParticleSystem::registerPreset(name, preset)` ‚Äî register by name. `emit(name, worldPos, intensity=1)` fills a `ParticleEmitterState` from the preset and appends to the per-frame list. `reset()` + `emitters()` follow the per-frame accumulate-then-read pattern.
- **GPU pool**: `kMaxParticles=8192` slots in a device-local SSBO (`GpuParticle` ‚Äî 80 bytes: pos+age, vel+maxAge, colorStart, colorEnd, sizeStart/sizeEnd/additive/_pad). Ring-buffer spawn: CPU writes up to `kMaxSpawnPerFrame=512` new particles to a per-frame host-visible staging buffer, `vkCmdCopyBuffer` into the pool.
- **Compute**: `particle_sim.comp` (local_size_x=64) decrements age, advances pos by vel√ódt, applies gravity. Dead particles (age‚â§0) written as zero and skipped in vertex shader.
- **Render**: `particle.vert` reads SSBO (set 0 binding 0) + camera UBO (set 0 binding 1); emits a 6-vertex camera-facing quad per instance (2 triangles). `particle.frag` outputs circular soft-edge colour with quadratic alpha falloff. Push constant `uint32_t renderAdditive` selects which particles each draw pass processes.
- **DamageDef.visualEffect hook**: `DamagePenalty::visualEffect` (string in `DamageDef`) is the preset name emitted by `SceneRenderer` via `EffectResolver` when an entity is damaged.

`engine/render/CameraController` produces a `CameraView` each frame. Three modes, switched via F1/F2/F4 in `main.cpp`:

- **Cockpit mode** (F1): camera at entity position, looking along entity forward + look offset. `setTarget(pos, ori)` + `setCockpitLook(yawDeg, pitchDeg)` each frame. RMB drag accumulates the look offset. Gimbal lock guard: substitutes up=`(1,0,0)` when view direction is near-parallel to world Y.
- **Chase mode** (F2): spherical orbit around entity position. Caller drives `setFreeOrbit(entityPos, yaw, pitch, radius)` each frame with orbit state updated from mouse LMB drag + scroll. Default: 25 m radius, 20¬∞ pitch, 180¬∞ yaw (behind entity).
- **Free mode** (F4, default): same `setFreeOrbit` orbit math, but pivot is a freely movable world-space reference point (WASD/QE pan, LMB orbit, scroll zoom, R reset). Camera is not attached to any entity.
- **Projection**: infinite reverse-Z perspective hand-built from `f = 1/tan(fovY/2)`: `proj[0][0]=f/aspect`, `proj[1][1]=-f` (Vulkan Y-flip), `proj[2][3]=-1`, `proj[3][2]=near`. `VkRenderer` reads `proj[3][2]` as the near-plane value for shadow cascade split.
- **`CameraView::worldOrigin`** is `glm::dvec3`. `VkRenderer` casts it to `vec3` only at GPU upload sites (shadow cascade center, particle origin).

### World terrain and theaters (feat/world-terrain)

The engine uses a single continuous **world terrain** rather than per-theater heightmap grids. Theaters are geographic regions (bounding boxes in world coordinates) within this single terrain, not separate maps. Players can fly anywhere in the world in one session; theater boundaries are mission conditions, not engine limits.

**Terrain ID:** `"world"` is the canonical ID. fl-base-pack provides global coverage; theater content packs override individual chunks at higher mod priority via `IContentPack::resolveTerrainChunk()`.

**Chunk format:** 16-bit grayscale PNG, 513√ó513 pixels, **15,360 m per chunk** (512 intervals √ó 30 m = native Copernicus GLO-30 resolution ‚Äî no upsampling needed). Three LOD levels: LOD 0 = 513√ó513, LOD 1 = 257√ó257, LOD 2 = 129√ó129. Chunk path convention: `terrain/<id>/lod<n>/chunk_<x>_<y>.png` (all lowercase, 4-digit zero-padded coordinates).

**LOD ring distances** (for a 15,360 m chunk): LOD 0 within 3√ó3 chunks (~46 km), LOD 1 within 5√ó5 (~77 km), LOD 2 within 7√ó7 (~107 km); evict beyond ~120 km. Maximum ~83 chunks in memory at steady state.

**`IContentPack::resolveTerrainChunk(terrainId, chunkX, chunkY, lod) ‚Üí optional<string>`**: pure virtual ‚Äî implemented in #172. `FolderContentPack` probes `terrain/<id>/lod<n>/chunk_<x>_<y>.png` under its mod dir (zero-padded 4-digit coords). `AssetManager::resolveTerrainChunk()` walks the priority stack; first-wins, not cached. `TerrainStreamer` calls this to resolve the path before queuing `IAsyncFilesystem::readFileAsync()`. `MockContentPack` in `test_content_system.cpp` has a configurable `chunkPaths` map (key format: `"terrainId:x:y:lod"`) for priority-stack tests; all other test mocks stub it as `return std::nullopt;`. **#131 added two more pure virtuals** (`getTrustLevel()` and `isNativePlugin()`) ‚Äî stubs needed in all 6 mock locations: `MockContentPack` + `BorrowedPack` in `test_content_system.cpp`, `EmptyContentPack` in `test_particle_system.cpp`, `MockContentPack` in `test_scene_renderer.cpp`, `NullContentPack` in `test_music_manager.cpp`, `MockTerrainPack` in `test_terrain_streamer.cpp`.

**`TerrainStreamer`** (`engine/render/TerrainStreamer.h`): manages chunk lifecycle. Constructor: `TerrainStreamer(TerrainManifest, AssetManager&, IAsyncFilesystem&, IRenderer*)` ‚Äî `IRenderer*` nullable for headless. Inherits `IAsyncFilesystemHandler`; registers/deregisters itself as the sole async FS handler. `update(dvec3 cameraPos)` computes desired chunk set (LOD rings), evicts out-of-range chunks, queues async reads or generates procedural fallback (rate-limited to `kMaxProceduralPerUpdate=8` chunks/frame to avoid first-frame hitch). `getRenderItems(dvec3 worldOrigin) ‚Üí vector<RenderItem>` returns best-available LOD per (cx,cy) for the 7√ó7 ring. `heightAt(double x, double z) ‚Üí double` bilinear interpolation from the LOD0 chunk only; returns 0.0 if not loaded. `surfaceAt()` stub returns 0. `chunkCount() ‚Üí size_t` exposed for tests. LOD ring semantics: each LOD maintained for its full concentric square ‚Äî LOD 0: 3√ó3 = 9 entries, LOD 1: 5√ó5 = 25 entries, LOD 2: 7√ó7 = 49 entries; **83 total** (9+25+49), all independent, enabling seamless LOD transitions. Mesh generation uses `buildTerrainMeshGlb()` (see `TerrainMeshBuilder.h`). Single shared `m_terrainMat` for all chunks. Wired in game `main.cpp` via `sceneRenderer.setTerrainStreamer(&ts)` and `p.asyncFilesystem->service()` + `ts.update()` each frame. In `fl-server`, wired headlessly: `asyncFilesystem->service()` + `terrainStreamer.update()` called from the main admin loop (~20 Hz); `m_renderer = nullptr` so GPU upload is skipped and only heightmap data is kept. PNG decode (for content-pack chunks) via `decodeTerrainChunkPng()` in `TerrainChunkIO.cpp` ‚Äî `STB_IMAGE_STATIC` already defined there to prevent ODR conflict with `VkResources.cpp`.

**Terrain rendering:** chunks are submitted as standard `RenderItem`s via `IRenderer::createMesh()`/`createMaterial()` ‚Äî no new `IRenderer` pure virtuals, no `FrameScene` changes, no `MockRenderer` updates required for Phase 2.

**fl-server** initializes `SDL3Filesystem` + `SDL3AsyncFilesystem` + `ModLoader` + `AssetManager` + headless `TerrainStreamer` (null renderer, #174) to serve `heightAt()` for authoritative physics. CMake deps added: `engine-render engine-content platform-sdl3`. Content packs are loaded via `ModLoader` from the `mods/` subdirectory of the server's working directory; the builtin procedural terrain (FBM, ~550 m base) is used when no content packs are present. The 5 sandbox entities in `main.cpp` spawn at `heightAt(x, z) + 500 m AGL`. **Threading constraint**: `TerrainStreamer::heightAt()` is main-thread-only; `WorldBroadcaster::onConnect` (sim thread) still uses hardcoded `t.pos[1] = 2000.0` ‚Äî making the spawn height thread-safe is tracked in #252.

**DEM source:** Copernicus GLO-30 (ESA, 30 m global, free). `tools/gen_terrain_chunks.py` (#176) converts GeoTIFF ‚Üí chunk PNGs at all 3 LOD levels. fl-base-pack `scripts/build_terrain.sh` downloads tiles from AWS Open Data (`s3://copernicus-dem-30m/`) and invokes the tool.

**Wire protocol:** fully documented in `docs/network-protocol.md`. `kProtocolVersion = 1`; `MsgHello` is 4 bytes (first reliable packet on connect; client disconnects on version mismatch); `MsgEntityEntry::pos[3]` is `double` (68 bytes); `MsgConnectAck` is 12 bytes (includes `assignedEntityIdx/Gen`); `MsgClientInput` is 44 bytes (client‚Üíserver, reliable); `MsgWeatherState` is 20 bytes (`MsgId::WeatherState=0x04`, unreliable ~6 Hz, additive ID); `MsgServerNotice` is 64 bytes (`MsgId::ServerNotice=0x05`, reliable, server‚Üíclient shutdown countdown; additive ID); `MsgAdminCommand` is 128 bytes (`MsgId::AdminCommand=0x06`, client‚Üíserver reliable; additive ID); `MsgAdminResponse` is 128 bytes (`MsgId::AdminResponse=0x07`, server‚Üíclient reliable; additive ID); `MsgLanBeacon` is 74 bytes (`MsgId::LanBeacon=0x10`, raw UDP broadcast, not ENet ‚Äî received by `DiscoveryListener`). Always parse with `std::memcpy`. #92 (version negotiation), #142 (authoritative game protocol), #170 (double-precision positions), #91 (LAN discovery), #39 (weather/time-of-day), #90 (shutdown countdown), and #229 (network admin channel) are all merged.

- `engine/content/AssetValidator.h` ‚Äî `AssetValidator::validate(type, header, totalSize) ‚Üí ValidationResult`; magic-byte checks (glTF binary/JSON, KTX2/PNG, OggS, Lua source/bytecode) + `ValidationLimits` size caps; called from `AssetManager::loadAsset` before caching; discards invalid assets and tries next pack (returns nullptr on all-pack failure)
- `engine/script/LuaSandbox.h` ‚Äî `LuaSandbox::create(packRootDir)` factory; `loadScript(source) ‚Üí bool`; deny-lists `io`, `os`, `package`, `debug`, `dofile`, `loadfile`; custom `require` restricted to `packRootDir/ai/<module>.lua`; rejects precompiled bytecode (`\x1b`); RAII destructor calls `lua_close()` (critical for ASAN `detect_leaks=1`); part of `engine-script` CMake library (links `lua::lua`)
- `cmake/dependencies.cmake` ‚Äî all FetchContent declarations; GLM is unconditional, Vulkan-specific deps are gated on `Vulkan_FOUND`; Lua 5.4 added (system-preferred `find_package(Lua 5.4 QUIET)` ‚Üí creates `lua::lua` INTERFACE target; FetchContent fallback via `FetchContent_Populate(lua_src)` for Lua repos without CMakeLists.txt ‚Äî same `FetchContent_Populate` pattern as `stb`; FetchContent target sets `COMPILE_WARNING_AS_ERROR OFF` to prevent Lua C sources from inheriting `-Werror`)
- `.github/actions/install-linux-deps/action.yml` ‚Äî composite action that installs Ubuntu apt deps for CI; boolean inputs `vulkan`, `gcc`, `clang`, `clang_format`, `python_tools`, `lcov`, `lua` (all default false); `lua: 'true'` installs `liblua5.4-dev`; used by ci.yml, asan.yml, coverage.yml (NOT by codeql.yml or release.yml which use inline apt-get directly)
- `platform/vulkan/VkRendererFactory.h` ‚Äî thin factory header; only include needed by game/tools to instantiate the renderer
- `tools/blender_gen.py` ‚Äî headless Blender 4.x parametric aircraft mesh generator; `blender --background --python tools/blender_gen.py -- --id <id> --output-dir <dir> [--wing-style delta|swept|straight] [--lod] [--bake-textures]`; outputs `<id>.glb` (clean + `_b` node), `<id>_dmg.glb`, optional LODs + PNGs; `.ktx2` URIs pre-wired in GLB JSON for tex-compress
- `tools/gen_terrain_chunks.py` ‚Äî GDAL-based terrain chunk pipeline (#176); converts any GeoTIFF/VRT to 16-bit grayscale PNG chunks at `terrain/<id>/lod<n>/chunk_<xxxx>_<yyyy>.png`; auto-detects UTM CRS from centroid (or `--srs EPSG:NNNNN`); 3 LOD levels via strided subsampling; parallel via `ProcessPoolExecutor` (`--workers N`); `--write-manifest` outputs `terrain/<id>.json`; `--skip-existing` for resumable planet-scale runs. Install: `apt install python3-gdal`. Tests: `tests/test_gen_terrain_chunks.py`.
